# 登录系统详细实现计划（更新版）
## 一、当前代码分析（更新）
### 1. 登录页面 (/src/app/login/page.tsx)
- 功能 ：实现用户登录表单，支持邮箱、密码输入和"记住我"功能
- 流程 ：
  - 从 localStorage 加载保存的邮箱
  - 从 URL 参数获取 redirect URL
  - 调用 login 函数进行登录
  - 登录成功后跳转到 redirect URL
  - 登录失败显示错误信息
### 2. 注册页面 (/src/app/register/page.tsx)
- 功能 ：实现用户注册表单，支持姓名、邮箱、电话、密码输入
- 问题 ：
  - 直接调用 API 注册，没有使用 AuthContext.tsx 中的 register 函数
  - 注册成功后跳转到登录页面，而不是直接登录
### 3. 购物车 (/src/lib/contexts/CartContext.tsx)
- 功能 ：管理购物车状态，支持添加、删除、更新商品
- 流程 ：
  - 未登录时使用 cart_guest 存储购物车
  - 登录后合并访客购物车到用户购物车
  - 未登录时点击购买会跳转到登录页面，并携带 redirect 参数
### 4. 认证上下文 (/src/lib/contexts/AuthContext.tsx)
- 功能 ：管理用户认证状态，提供登录、注册、登出功能
- 问题 ：
  - 期望 API 返回 access_token 和 refresh_token
  - 使用 Bearer 令牌认证
  - 注册函数未被使用
### 5. 登录 API (/src/app/api/auth/login/route.ts)
- 功能 ：处理登录请求，验证账号密码
- 问题 ：
  - 返回 token 而不是 access_token
  - 使用 bcrypt.compare 验证密码，但数据库中的密码可能是明文
  - 生成的是简单随机字符串，不是标准 JWT
### 6. 注册 API (/src/app/api/auth/register/route.ts)
- 功能 ：处理注册请求，创建新用户
- 优点 ：
  - 已经使用 JWT 生成令牌
  - 返回 access_token 和 refresh_token
  - 使用 bcrypt.hash 加密密码
- 问题 ：
  - 缺少对应的前端调用
### 7. 缺失的 API 路由
- /api/auth/me ：用于验证令牌和获取用户信息
- /api/auth/refresh ：用于刷新令牌
- /api/auth/logout ：用于登出用户
### 8. 相关页面
- 需要登录状态的页面 ：
  - account/page.tsx ：个人中心
  - cart/page.tsx ：购物车
  - checkout/page.tsx ： checkout
  - checkout/success/page.tsx ： checkout 成功
  - customize/page.tsx ： 定制服务
## 二、登录系统改进计划（更新）
### 1. 后端 API 改进
- 修改登录 API 响应格式 ：
  
  - 将 token 改为 access_token
  - 添加 refresh_token
  - 使用 JWT 生成令牌，与注册 API 保持一致
  - 确保响应格式与 AuthContext.tsx 期望一致
- 密码验证逻辑 ：
  
  - 检查数据库中密码的存储方式
  - 对于明文密码，直接比较
  - 对于哈希密码，使用 bcrypt.compare 验证
  - 考虑将所有明文密码更新为哈希密码
- 添加缺失的 API 路由 ：
  
  - /api/auth/me ：验证令牌并返回用户信息
  - /api/auth/refresh ：使用 refresh_token 刷新 access_token
  - /api/auth/logout ：处理登出请求
- 令牌配置 ：
  
  - access_token 过期时间：2 小时
  - refresh_token 过期时间：30 天
  - 使用环境变量存储 JWT 密钥
### 2. 前端登录流程改进
- 登录页面 ：
  
  - 保持现有功能
  - 确保登录成功后跳转到正确的页面
- 注册页面 ：
  
  - 修改为使用 AuthContext.tsx 中的 register 函数
  - 注册成功后直接登录，而不是跳转到登录页面
  - 登录成功后跳转到 /account 或之前的页面
- 登录状态管理 ：
  
  - 确保 AuthContext.tsx 正确处理登录状态
  - 修复令牌存储和读取逻辑
  - 确保登录状态在页面刷新后保持
- 令牌过期处理 ：
  
  - 实现令牌过期时的自动刷新机制
  - 刷新失败时跳转到登录页面
### 3. 业务逻辑集成
- 导航栏 ：
  
  - 根据登录状态显示不同的菜单
  - 登录状态显示用户头像和个人中心链接
  - 未登录状态显示登录和注册链接
- 购物车 ：

  - 登录后合并访客购物车到用户购物车
  - 确保购物车状态在登录前后保持一致
- 购买流程 ：
  
  - 未登录时点击购买跳转到登录页面
  - 登录成功后返回商品详情页
  - 确保购买流程的完整性
- 个人中心 ：
  
  - 未登录时跳转到登录页面
  - 登录状态显示用户信息和相关功能
  - 确保用户信息正确显示
- 权限控制 ：
  
  - 根据用户角色显示不同的功能
  - 确保管理员功能只对管理员开放

## 三、技术实现要点（更新）
### 1. 后端实现
- 使用 bcrypt 加密密码 ：
  
  - 确保新注册用户的密码使用 bcrypt 加密
  - 考虑将数据库中的明文密码更新为哈希密码
- 使用 jsonwebtoken 库生成 JWT ：
  
  - 生成标准的 JWT 令牌
  - 设置合理的过期时间
  - 包含必要的用户信息
- 实现令牌验证和刷新 ：
  
  - 验证令牌的有效性
  - 实现令牌刷新机制
  - 处理令牌过期的情况
- 添加必要的 API 路由 ：
  
  - 实现 /api/auth/me 路由
  - 实现 /api/auth/refresh 路由
  - 实现 /api/auth/logout 路由
### 2. 前端实现
- 正确处理登录状态 ：
  
  - 存储和读取令牌
  - 处理登录状态的变化
  - 确保登录状态在页面刷新后保持
- 实现登录成功后的跳转 ：
  
  - 从 URL 参数获取 redirect URL
  - 跳转到正确的页面
  - 处理默认跳转
- 处理未登录状态 ：
  
  - 检测未登录状态
  - 跳转到登录页面
  - 传递 redirect 参数
- 注册流程改进 ：
  
  - 使用 AuthContext.tsx 中的 register 函数
  - 注册成功后直接登录
  - 跳转到合适的页面
### 3. 安全性考虑
- 密码加密 ：确保所有密码都使用 bcrypt 加密存储
- 令牌安全 ：使用 JWT 令牌，并设置合理的过期时间
- HTTPS ：确保所有 API 请求使用 HTTPS
- 输入验证 ：在前端和后端都进行输入验证，防止恶意输入
- 环境变量 ：使用环境变量存储 JWT 密钥，避免硬编码
