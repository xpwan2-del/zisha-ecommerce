#### H-001：支付模块缺少统一响应格式

**文件**：`/src/app/api/payments/paypal/route.ts`  
**位置**：第 296-299 行  
**类型**：API设计 + 规范遵循

**问题描述**：
PayPal 支付创建成功后，返回格式与其他 API 不一致：

```typescript
// 当前代码
return NextResponse.json({
  id: paypalResponse.id,
  url: approvalUrl
});

// 应改为（遵循统一格式）
return NextResponse.json({
  success: true,
  data: {
    id: paypalResponse.id,
    url: approvalUrl
  }
});
```

**影响范围**：
- `/api/payments/paypal/route.ts`
- `/api/payments/alipay/route.ts`
- `/api/payments/stripe/route.ts`

**修复建议**：
所有支付创建 API 应返回统一格式：
```typescript
return NextResponse.json({
  success: true,
  data: {
    payment_id: paypalResponse.id,
    redirect_url: approvalUrl
  }
});
```

#### H-002：Stripe API 缺少用户认证

**文件**：`/src/app/api/payments/stripe/route.ts`  
**位置**：第 76-154 行  
**类型**：安全漏洞

**问题描述**：
Stripe 支付创建接口没有验证用户是否登录，任何人都可以调用此接口创建支付。

```typescript
// 当前代码 - 没有认证检查
export async function POST(request: NextRequest) {
  try {
    const body = await request.json();
    // 直接处理支付，没有 requireAuth()
```

**安全风险**：
- 未登录用户可创建支付订单
- 可能被恶意利用创建大量无效支付会话

**修复建议**：
添加用户认证：
```typescript
export async function POST(request: NextRequest) {
  const authResult = requireAuth(request);
  if (authResult.response) {
    return authResult.response;
  }
  // 继续处理...
}
```

#### H-004：数据库并发写入问题

**文件**：`/src/lib/db.ts`  
**位置**：第 130-153 行  
**类型**：性能 + 数据一致性

**问题描述**：
每次写入操作都会同步写入磁盘，在高并发场景下会成为性能瓶颈。

```typescript
// 当前代码
database.run(sql, params);
const changes = database.getRowsModified();
// ...
fs.writeFileSync(dbPath, buffer);  // 同步写入
```

**影响**：
- 并发写入时可能导致数据丢失
- 响应延迟增加

**修复建议**：
- 使用批量写入 + 定期刷新
- 或使用 WAL 模式提高并发性能
- 参考 Turso 的异步同步机制

#### H-005：硬编码汇率

**文件**：`/src/app/api/quick-order/create/route.ts`  
**位置**：第 353-354 行  
**类型**：业务逻辑

**问题描述**：
汇率硬编码在代码中，应从配置或数据库获取。

```typescript
// 当前代码
const totalUSD = finalAmount * 0.2722;  // AED -> USD
const totalCNY = finalAmount * 1.9558;   // AED -> CNY
```

**风险**：
- 汇率变动需修改代码
- 不同货币结算金额可能不准确

**修复建议**：
```typescript
// 从配置或数据库获取
const rates = await query('SELECT * FROM exchange_rates WHERE base = ?', ['AED']);
const totalUSD = finalAmount * rates.rows[0].usd_rate;
```

#### M-003：优惠券 API 使用 Header 传用户ID

**文件**：`/src/app/api/coupons/route.ts`  
**位置**：第 66、227 行

**问题描述**：
使用 `x-user-id` header 传递用户ID，而不是使用 JWT Token 验证。

```typescript
// 当前代码
const userId = request.headers.get('x-user-id');
```

**安全风险**：
- 用户可伪造任意用户ID
- 权限绕过漏洞

**修复建议**：
```typescript
// 使用标准的认证方式
const authResult = requireAuth(request);
if (authResult.response) return authResult.response;
const userId = authResult.user.userId;
```

#### M-005：库存扣减缺少事务保护

**文件**：`/src/app/api/inventory/reserve/route.ts`  
**位置**：第 239-246 行

**问题描述**：
库存扣减操作没有使用事务，可能导致数据不一致。

```typescript
// 当前代码
await query(
  'UPDATE inventory SET quantity = quantity - ? ...',
  [...]
);
await query(`INSERT INTO inventory_transactions (...)`);  // 如果失败，库存已扣但无流水
```

**修复建议**：
```typescript
await query('BEGIN TRANSACTION');
try {
  await query('UPDATE inventory SET ...');
  await query('INSERT INTO inventory_transactions ...');
  await query('COMMIT');
} catch (error) {
  await query('ROLLBACK');
  throw error;
}
```
