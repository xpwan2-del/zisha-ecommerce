## 二、数据库问题

### 2.1 高优先级问题

| 问题 | 文件/位置 | 影响 |
|------|----------|------|
| 外键约束缺失 | orders, order_items, inventory 等核心表 | 数据完整性风险 |
| products.stock 冗余 | products 表 | 与 inventory 表数据不一致 |
| 订单号可能重复 | orders.order_number | 高并发下订单号重复风险 |
| user_coupons 唯一约束 | user_coupons 表 | 同一用户可重复领取优惠券 |

### 2.2 中优先级问题

| 问题 | 影响 |
|------|------|
| 索引缺失 | 查询性能下降 |
| transaction_type 字段混用 | 代码维护困难 |
| 状态字段缺少 CHECK 约束 | 可能存入非法状态值 |
| 字段类型不统一 | created_at 使用 TIMESTAMP/DATETIME 混用 |

### 2.3 建议的数据库修复

```sql
-- 1. 添加缺失的外键约束
ALTER TABLE orders ADD CONSTRAINT fk_orders_users 
  FOREIGN KEY (user_id) REFERENCES users(id);

ALTER TABLE order_items ADD CONSTRAINT fk_order_items_orders 
  FOREIGN KEY (order_id) REFERENCES orders(id);

ALTER TABLE order_items ADD CONSTRAINT fk_order_items_products 
  FOREIGN KEY (product_id) REFERENCES products(id);

ALTER TABLE inventory ADD CONSTRAINT fk_inventory_products 
  FOREIGN KEY (product_id) REFERENCES products(id);

-- 2. 删除冗余的 products.stock 字段（可选）
-- ALTER TABLE products DROP COLUMN stock;

-- 3. 添加必要的索引
CREATE UNIQUE INDEX idx_users_email ON users(email);
CREATE UNIQUE INDEX idx_orders_order_number ON orders(order_number);
CREATE INDEX idx_orders_user_status ON orders(user_id, order_status);
CREATE UNIQUE INDEX idx_cart_user_product ON cart_items(user_id, product_id);
CREATE UNIQUE INDEX idx_user_coupons_user_coupon ON user_coupons(user_id, coupon_id);

-- 4. 添加 CHECK 约束
ALTER TABLE orders ADD CONSTRAINT chk_order_status 
  CHECK (order_status IN ('pending', 'paid', 'shipped', 'completed', 'cancelled'));
```
