# 数据库操作规则

## 核心规则

**修改数据库前必须通知用户：**
1. 说明要做什么操作
2. 说明会影响哪些表和数据
3. 等待用户确认后才能执行
4. 执行后报告修改结果

## 禁止事项

- 不通知就修改数据库 ❌
- 修改前不说明影响范围 ❌
- 修改后不报告结果 ❌

## 风险操作示例

| 操作 | 必须通知 |
|-----|---------|
| ALTER TABLE | 必须 |
| DROP TABLE | 必须 |
| DELETE 数据 | 必须 |
| INSERT 数据 | 必须|
| UPDATE 数据 | 必须 |

## 通知模板

```
我需要：[操作描述]
会影响：[表名/数据]
请确认是否继续？
```
