## 二、订单列表页面（改进版）

### 2.1 数据展示规范

订单列表只展示关键概览信息，详细信息在详情页展示。

| 数据项 | 示例值 | 说明 |
|-------|-------|------|
| 订单号 | ORD177718446567514 | 订单唯一标识 |
| 下单时间 | 2026-04-26 14:30 | 格式：YYYY-MM-DD HH:mm |
| 订单状态 | **待付款** | 不同状态不同颜色标签 |
| 商品缩略图 | [图片] 80x80px | 第一张商品图 |
| 商品名称 | 竹节紫砂壶 (+2件) | 多商品显示"+X件" |
| 订单金额 | ¥1,313.25 AED | 最终支付金额 |
| 操作按钮 | 立即支付 / 再次购买 | 根据状态显示 |
| 查看详情 | → | 跳转到详情页 |

### 2.2 不展示的数据

以下数据只在详情页展示，列表页不展示：
- ❌ 商品规格/属性
- ❌ 收货地址详情
- ❌ 支付方式
- ❌ 物流信息
- ❌ 费用明细拆解

## 四、实现规范

### 4.1 Next.js 实现规范

| 规范 | 说明 |
|-----|------|
| 图片 | 使用 `fill` + `object-cover` 实现响应式图片 |
| 字体 | 在 layout.tsx 中全局设置，不在页面单独导入 |
| 加载 | 使用 Skeleton 占位，防止布局跳动 |
| 图标 | 使用 Heroicons/Lucide SVG，不使用 emoji |

### 4.2 禁止事项

| 禁止事项 | 正确做法 |
|---------|---------|
| ❌ 使用 emoji 作为图标 | ✅ 使用 Heroicons/Lucide SVG |
| ❌ 前端做业务逻辑 | ✅ 所有逻辑在服务端完成 |
| ❌ 修改现有页面布局 | ✅ 使用条件渲染添加新功能 |
| ❌ 使用 mock 数据 | ✅ 从 API 获取真实数据 |
